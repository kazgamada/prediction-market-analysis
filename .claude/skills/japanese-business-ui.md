# 日本語業務 UI 規約

## 概要

日本のビジネス文脈で使われる業務アプリケーション特有の UI 規約・入力パターンをまとめる。海外製の UI フレームワークのデフォルトではフィットしない部分をここで定義する。

## 使用タイミング

- 顧客管理・案件管理・申込フォームなど業務画面を作るとき
- 請求書・見積書・領収書など帳票を作るとき
- 住所・電話番号・日付・金額の入力 UI を作るとき

### 氏名

- 姓・名は別フィールドに分割（並び順: 姓 → 名）
- ふりがな（カナ）フィールドを併設（入力時自動変換 or 必須入力）
- 敬称: デフォルトは「様」。社内向けは「さん」。

### 住所

- 郵便番号 → 住所自動補完（例: `zipcloud` API、`@actbase/react-daum-postcode` の日本版相当）
- フィールド分割: 郵便番号 / 都道府県 / 市区町村 / 番地 / 建物名
- 「建物名」は任意（集合住宅でなければ空）

### 日付

- 入力: `YYYY/MM/DD` または `YYYY-MM-DD` を許容
- 表示: 業務帳票は `YYYY年MM月DD日`、一覧は `YYYY/MM/DD`
- 和暦（令和）表示は **明示指定があるときのみ**。デフォルトは西暦
- タイムゾーンは Asia/Tokyo で固定（Supabase に保存するときは UTC）

### 金額

- 表示: `¥1,234,567`（3桁カンマ、円記号前）
- 入力: 半角数字のみ受け付け、カンマ自動挿入
- 税表示: 税込 / 税抜を明示。デフォルトは税込

### 電話番号

- フィールドは1つでよい（半角数字 + ハイフン）
- 自動ハイフン挿入は任意（入力速度を落とすことがあるので案件ごとに判断）

### 全角・半角

- 英数字は原則半角に自動変換
- 氏名の漢字・かなはそのまま
- 郵便番号・電話番号・金額は入力時に半角化

### 帳票 PDF

- 日本語フォント埋め込み必須（Noto Sans JP など）
- 縦書き・横書きは案件による（請求書は横書きが多い）

### 押印 UI

- 電子印影をアップロードして配置するパターン
- 実印が必要な場合は紙運用を並行
